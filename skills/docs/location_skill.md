# Location Skill - GPS 定位服务

## 描述
获取设备当前位置信息，支持 IP 定位、GPS 坐标获取。当用户询问"附近"相关内容时，此技能用于获取当前位置坐标。

## 适用场景
当用户需要以下信息时调用此技能：
- 获取当前 GPS 位置
- 查询 IP 对应的大致位置
- 需要定位才能提供的服务（如查找附近餐厅）
- 用户说"附近"但没有提供具体位置时

## 工具函数

### get_current_location

获取当前设备的地理位置信息。优先使用 GPS，如果不支持则使用 IP 定位。

**参数：** 无

**返回：**
- 经纬度坐标
- 城市信息
- 国家和省份

**使用示例：**
```python
get_current_location()
```

**返回示例：**
```
📍 当前位置信息：

坐标: 121.4737,31.2304
城市: 中国 上海市 上海市

💡 提示：这是基于 IP 的大致位置。
```

### get_location_by_ip

通过 IP 地址获取当前设备的大致位置。与 get_current_location 功能相同。

### get_gps_coordinates

获取当前 GPS 坐标。由于 Python 脚本运行在服务器端，无法直接获取客户端 GPS，会返回常见城市坐标参考。

### parse_address_from_coords

从经纬度坐标解析地址信息（简化版，详细解析请使用 gaode 技能）。

**参数：**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| longitude | float | 是 | 经度 |
| latitude | float | 是 | 纬度 |

## 使用场景示例

### 场景 1：用户询问附近餐厅
```
用户: 附近有什么麦当劳？
系统: 需要获取您的位置
→ 调用 location.get_current_location()
→ 获取坐标: 121.4737,31.2304
→ 调用 gaode.search_poi(keywords="麦当劳", city="上海")
```

### 场景 2：用户没有提供位置
```
用户: 帮我找找附近的加油站
系统: 检测到"附近"但没有位置信息
→ 调用 location.get_current_location()
→ 获取位置后推荐附近加油站
```

## 注意事项
- IP 定位精度有限，可能只能定位到城市级别
- 如需精确位置，建议用户手动提供坐标
- GPS 定位需要在支持 GPS 的设备上运行

## 相关技能
- **gaode**: 高德地图服务，用于详细的位置解析和周边搜索
- **mcdonalds**: 麦当劳服务，结合 location 可以推荐附近餐厅
